A Shipfox trigger has a `source` and an `event`. This page lists every source and event supported today, along with any source-specific requirements.

## manual

The built-in on-demand source. Use `manual` when you want to start a workflow from the Shipfox dashboard or API without an external integration.

- **Events:** `fire`
- **Integration required:** None.
- **Limit:** At most one `manual` trigger per workflow.

```yaml
triggers:
  on_demand:
    source: manual
    event: fire
```

## github

The GitHub integration source. Use `github` to start a workflow when repository events occur.

- **Events:** `push`
- **Integration required:** A GitHub repository connected to the project.
- **Branch ref format:** `event.ref` is the bare branch name (e.g., `main`), not the full ref path `refs/heads/main`.
- **Filter note:** The `filter` field parses but is not yet evaluated — the trigger fires on every `push` event regardless of filter content.

```yaml
triggers:
  on_push:
    source: github
    event: push
```

## sentry

The Sentry integration source. Use `sentry` to start a workflow when Sentry issues change state.

- **Events:** `issue.created`, `issue.resolved`, `issue.assigned`, `issue.archived`, `issue.unresolved`
- **Integration required:** A Sentry integration configured in workspace settings.

```yaml
triggers:
  on_issue:
    source: sentry
    event: issue.created
```

<Note>
  GitLab, Linear, Slack, and schedule triggers are planned. Today, use `source: manual` for sources that aren't yet supported.
</Note>

## filter (not enforced)

The `filter` field accepts a CEL expression string. It is intended to narrow which events fire the trigger — for example, `event.ref == "main"` on a GitHub push trigger would ideally restrict the trigger to pushes on the `main` branch only.

Today, `filter` is parsed and stored alongside the trigger definition but is **not evaluated at runtime**. The trigger fires on every matching `(source, event)` pair regardless of what the filter expression contains. Authoring a filter value is safe and will take effect once evaluation is enabled — no schema changes will be required.
